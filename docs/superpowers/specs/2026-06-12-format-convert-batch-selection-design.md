# 格式转换批量选择设计补充

## 目标

在现有格式转换页面上补一版可运行的批量转换工作台：

- 来源模式仍然只有一种，用户先选 `本地上传` 或 `OpenList 云文件`
- 默认来源模式为 `本地上传`
- 当前来源模式内支持多选文件
- 点击一次提交后，前端为每个源文件分别创建一条独立转换任务
- 多文件模式下不允许逐个自定义输出文件名，统一按规则自动生成

## 页面结构

- 顶部保留来源模式切换，默认选中本地
- 中间主区域只渲染当前模式的选择器
  - 本地：Semi `Upload` 拖拽多选
  - 云端：Semi `Tree` 展示目录结构，叶子文件节点提供复选框
- 右侧或下方展示“已选文件”篮子，支持移除单项和清空
- 下方保留统一转换参数区和提交区

## 云端选择规则

- 允许跨目录多选文件
- 不支持把目录当作转换源直接提交
- `Tree` 用于目录浏览与文件勾选
- 目录节点负责展开/折叠与异步加载
- 文件节点由前端维护独立勾选状态，并同步到“已选文件”篮子

## 命名规则

- 单文件模式下保留现有自定义输出文件名能力
- 多文件模式下输出文件名锁定为自动生成
- 自动命名格式：
  - `原文件名_输出格式`
  - 若存在非 `source` 分辨率，追加 `_分辨率`
  - 若存在且与输出格式不同的音频编码，追加 `_音频编码`
- 示例：
  - `movie.mov -> movie_mp4_1080p_aac.mp4`
  - `demo.mkv -> demo_webm_720p_opus.webm`

## 提交规则

- 本地多选：按文件顺序逐个上传并创建任务
- 云端多选：按已选文件顺序逐个创建云转换任务
- 每个任务独立出现在现有任务列表中
- 任务之间不共享重试、清理和状态
