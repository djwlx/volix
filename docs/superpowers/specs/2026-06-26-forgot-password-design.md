# 忘记密码设计

目标是在 Volix 现有邮箱验证码和 SMTP 能力上，增加用户忘记密码后的自助重置能力，并允许邮件直接跳转到重置密码页面。

方案采用混合模式：
- 邮件中同时发送 6 位验证码和重置链接。
- 后端支持两种重置方式：`email + verifyCode + newPassword`，或 `token + newPassword`。
- token 先使用内存态缓存，和现有注册验证码一致，优先做到可用。
- 前端登录页增加忘记密码入口与重置模式，支持用户手动输验证码，也支持通过邮件链接自动带参进入。

边界：
- 不新增数据库表。
- 不改用户登录 token 逻辑。
- SMTP 仍复用系统配置中的注册邮箱 SMTP。
- 服务重启后，重置 token 和验证码会失效，这是当前阶段接受的限制。
