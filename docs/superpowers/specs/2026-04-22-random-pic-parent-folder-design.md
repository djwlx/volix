# 随机图片增加“同文件夹随机”能力设计

## 背景

当前 `随机图片` 页面只支持从已缓存的 115 图片目录中全局随机一张图片。用户在看到一张满意的图片后，无法继续围绕这张图片所属的父文件夹浏览相近内容，也无法直观看到当前图片所在的完整路径。

这次改动目标是：

- 在当前图片基础上，新增“同文件夹随机”按钮。
- 同文件夹随机时，范围限定为当前图片父文件夹。
- 页面展示当前图片的完整路径，便于确认来源目录。

## 用户体验

### 页面行为

随机图片页面保持现有主视觉不变，在底部操作区增加一项与上下文相关的浏览能力：

- `换一张`：保持现有行为，仍然从所有可用缓存图片中全局随机。
- `同文件夹随机`：基于当前图片的父文件夹随机下一张图片。
- `喜欢`：保持现有行为。

按钮区上方展示当前图片完整路径，默认显示完整字符串，不做折叠裁剪策略设计变更；如果样式层面需要兼顾移动端，可使用现有布局下的自动换行。

### 边界交互

- 如果当前父文件夹下只有一张图，点击 `同文件夹随机` 后保持当前图片不变，并提示用户当前目录没有其他图片可切换。
- 如果当前图片缺少父文件夹上下文，`同文件夹随机` 按钮禁用。
- 如果按父文件夹随机请求失败，保持当前图片不变，并给出错误提示。

## 后端设计

### 返回模型扩展

现有 `/115/pic?mode=json` 返回图片地址、`cid`、`pc` 等最小元信息。本次需要扩展返回内容，补齐前端展示和后续随机所需的上下文：

- `path`：当前图片完整路径。
- `parentPath`：当前图片父文件夹完整路径。

现有字段保持兼容，前端仍可继续依赖 `url`、`cid`、`pc`。

### 新增接口

新增按父文件夹随机图片接口，建议保持在 `115` 模块下，语义独立于全局随机接口：

- 路由：`GET /115/pic/parent-random`
- 输入：当前图片 `pc`
- 输出：与 `/115/pic?mode=json` 对齐的图片元信息，并同样包含 `path`、`parentPath`

独立接口优于在原接口上堆叠模式参数，因为：

- 前端调用意图更清晰。
- 控制器和服务层职责边界更明确。
- 后续如果继续增加“同根目录随机”或“同收藏目录随机”等语义，不会让单一路由变得混杂。

### 服务层逻辑

新增按父文件夹随机的服务能力，核心流程如下：

1. 根据传入 `pc` 查出当前图片记录。
2. 从记录中读取 `cid` 与 `parentCid`。
3. 在同一 `cid + parentCid` 范围内查询所有候选图片。
4. 尽量排除当前图片自身后随机选择一张。
5. 取到目标图片后，复用现有文件详情获取逻辑组装直链与文件名。
6. 返回完整元信息与路径上下文。

### 路径生成

当前数据库缓存中已有图片记录的 `cid`、`parentCid`、`name`、`pc`。为了生成完整路径，需要能够从当前图片向上恢复目录链。

推荐做法：

- 复用或补充文件数据库查询能力，根据目录 `cid` 查目录节点信息。
- 从当前图片父目录向上追溯直到缓存根目录。
- 拼出 `parentPath`，再追加当前文件名生成 `path`。

这里不要求引入新的长期缓存结构，优先基于现有表和查询方法补足能力。

### 异常与约束

- 未传 `pc` 或查不到当前图片时，返回参数错误或资源不存在错误。
- 当前父文件夹没有任何图片时，返回明确错误。
- 当前父文件夹只有当前这一张图时，不抛异常，返回当前图并附带“无其他图片”语义提示，或由服务层抛出可识别错误、前端转为提示。推荐前者，这样前端状态最稳定。

## 前端设计

### 状态模型

当前前端 `PicMeta` 仅包含：

- `src`
- `cid`
- `pc`

本次扩展为：

- `src`
- `cid`
- `pc`
- `path`
- `parentPath`

### 交互流

- 首次进入页面，仍调用全局随机接口获取首张图。
- 点击 `换一张`，调用全局随机接口并替换当前状态。
- 点击 `同文件夹随机`，调用新接口并替换当前状态。
- 点击 `喜欢`，保持现有逻辑不变。

### 展示位置

完整路径展示放在图片底部按钮区上方，原因如下：

- 与当前图片操作上下文强相关。
- 不会打断图片主视觉区域。
- 可以自然服务于“同文件夹随机”按钮，让用户知道当前随机范围来自哪里。

## 测试设计

### 后端测试

优先补服务层单测，覆盖以下行为：

- 全局随机接口返回新增的 `path` 和 `parentPath`。
- 按父文件夹随机时，只会命中当前父文件夹下的候选图片。
- 当前父文件夹存在多张图片时，优先排除当前图片本身。
- 当前父文件夹只有一张图片时，返回当前图片并给出可识别结果。

### 前端测试

如果该页面已有测试基础，则补交互测试；如果没有，则至少保证类型与运行链路可通过。

重点验证：

- 页面能渲染完整路径。
- `同文件夹随机` 调用正确接口并更新图片。
- 接口失败时保持当前图片状态并提示错误。

## 实施范围

本次只处理 `随机图片` 页面与 `115` 图片服务，不扩展到 OpenList 随机图片能力，也不引入新的目录管理 UI。

## 风险与控制

- 路径恢复依赖当前缓存表是否保留足够的目录信息；如果目录链信息不足，需要在服务层补最小查询能力。
- 前端新增字段后需保持与旧返回兼容，避免页面在接口未完整返回时直接崩溃。
- “同文件夹随机”与“喜欢”按钮共享当前图片上下文，更新状态时要避免使用旧状态提交错误参数。

## 验收标准

- 页面新增 `同文件夹随机` 按钮。
- 点击后可以在当前图片父文件夹范围内切换图片。
- 当前图片完整路径在页面中可见。
- 父文件夹无其他图片时，页面不报错、不清空图片，并给出明确提示。
- 现有 `换一张` 与 `喜欢` 功能行为不回退。
